# 📊 تحليل النظام واقتراحات التطوير
## `org-dashboard` — Laravel + Livewire

---

## 🔍 ما يحتوي عليه النظام حالياً

| الوحدة | الوصف |
|--------|-------|
| **Activities** | إنشاء وإدارة الأنشطة الميدانية مع الحالة، التواريخ، القطاع، المنطقة |
| **Activity Parcels** | توزيع الطرود والمساعدات المرتبطة بالأنشطة |
| **Beneficiaries** | المستفيدون المرتبطون بكل نشاط |
| **Displacement Camps** | مخيمات النازحين مع إحداثيات GPS |
| **Operations Map** | خريطة تفاعلية تعرض الأنشطة والمخيمات |
| **Purchase Requisitions** | طلبات الشراء ومقارنة العروض |
| **Education Module** | مجموعات، طلاب، مناهج، حضور، معلمون |
| **Surveys** | استبيانات للطلاب مع مقاييس تقييم |
| **Reports** | تقرير شهري للمدير، سجل يومي، تحليل مالي، مستفيدون |
| **Gallery** | معرض صور المشاريع |
| **Calendar** | تقويم الفعاليات |
| **Employees** | إدارة الموظفين |
| **Partners** | المنظمات والشركاء |
| **AI Copilot** | مساعد ذكاء اصطناعي |

## 💡 المقترحات التطويرية (مرتبة حسب الأولوية)

---

### 🔴 الأولوية العالية — تكمل ما هو موجود

#### 1. 📍 تتبع تقدم الأنشطة (Activity Progress Tracker)
**المشكلة:** لا توجد طريقة لمعرفة نسبة الإنجاز الفعلية لكل نشاط.
**المقترح:** إضافة حقل `progress_percentage` (0-100%) أو `milestones` (مراحل رئيسية) لكل نشاط، مع شريط تقدم بصري (`progress bar`) في صفحة قائمة الأنشطة.
**الفائدة:** تحسين التخطيط ومتابعة الإنجاز الفعلي.

---

#### 2. 🔔 نظام الإشعارات والتنبيهات (Notification System)
**المشكلة:** لا يوجد نظام يُنبّه المستخدمين بالأحداث المهمة.
**المقترح:**
- تنبيهات عند اقتراب نهاية نشاط (`end_date` خلال 3 أيام)
- تنبيه عند إضافة نشاط جديد أو تغيير حالته
- تنبيه لأصحاب الطلبات عند تحديث حالة `Purchase Requisition`
- نظام `Notifications` داخل Laravel مع Bell Icon في الهيدر

---

#### 3. 💰 تتبع الميزانية والإنفاق الفعلي (Budget vs. Actual Tracking)
**المشكلة:** يوجد `cost` و`cost_nis` لكل نشاط لكن لا مقارنة مع ميزانية مخططة.
**المقترح:** إضافة `planned_budget` لكل نشاط أو قطاع، وتقرير يقارن بين المبالغ المخططة والمنفقة فعلياً مع رسم بياني.

---

#### 4. 📤 تصدير التقارير PDF (PDF Export)
**المشكلة:** التصدير حالياً فقط إلى Excel.
**المقترح:** إضافة تصدير PDF للتقارير الشهرية وتقرير النشاط الفردي باستخدام `barryvdh/laravel-dompdf`، مع تنسيق جميل يحتوي شعار المنظمة.

---

#### 5. 🗓️ تخطيط الأنشطة المستقبلية (Activity Planning Board)
**المقترح:** عرض Gantt Chart أو Kanban Board بسيط للأنشطة القادمة مصنفة حسب الحالة (Planned / In Progress / Completed)، مما يسهل التخطيط المستقبلي.

---

### 🟡 الأولوية المتوسطة — تحسينات جوهرية

#### 6. 👥 إدارة فرق العمل الميداني (Field Teams Management)
**الملاحظة:** يوجد موديل `ActivityWorkTeam` لكن لا يبدو أنه مفعّل بشكل كامل.
**المقترح:** صفحة لإدارة فرق العمل، تعيين أعضاء لكل نشاط، ومعرفة عبء العمل على كل موظف.

---

#### 7. 📸 ربط الصور بالمراحل (Photo Evidence per Milestone)
**المقترح:** بدلاً من رفع مرفقات عامة، ربط كل صورة بـ "مرحلة" أو "تاريخ" محدد، مما يسمح بسرد قصة مرئية لتقدم النشاط.

---

#### 8. 📊 لوحة تحكم تنفيذية (Executive Dashboard)
**المقترح:** صفحة Dashboard مخصصة للمدير تعرض:
- KPIs رئيسية (عدد الأنشطة، المستفيدون، الإنفاق هذا الشهر)
- رسوم بيانية تفاعلية (Chart.js أو ApexCharts)
- مقارنة بين الشهر الحالي والشهر الماضي

---

#### 9. 🤝 بوابة المستفيدين (Beneficiary Portal)
**المقترح:** صفحة خارجية (بدون تسجيل دخول) يتمكن المستفيد من خلالها من:
- التسجيل في البرنامج
- معرفة حالة طلبه
- استلام إشعار SMS/WhatsApp

---

#### 10. 📝 نظام التعليقات الداخلي (Internal Comments)
**الملاحظة:** يوجد موديل `ActivityComments` لكنه قد لا يكون مفعلاً في الواجهة.
**المقترح:** تفعيل قسم التعليقات الداخلية في صفحة تفاصيل كل نشاط، مع إمكانية ذكر المستخدمين `@mention` لإشعارهم.

---

### 🟢 الأولوية المنخفضة — ميزات مستقبلية

#### 11. 🌍 دعم متعدد اللغات (Multi-language)
**المقترح:** تفعيل اللغة العربية بشكل كامل كواجهة أساسية مع RTL، مع إمكانية التبديل بين العربية والإنجليزية.

---

#### 12. 📱 تحسين التجربة المحمولة (Mobile-First UX)
**المقترح:** مراجعة جميع الصفحات على الجوال، وإنشاء تجربة موحدة للمستخدمين في الميدان الذين يدخلون البيانات من هواتفهم.

---

#### 13. 🤖 تحليل ذكي للبيانات بالذكاء الاصطناعي
**المقترح:** توسيع ميزة AI Copilot الموجودة لتشمل:
- تلخيص تقرير شهري تلقائياً بالعربية
- اقتراح أفضل المناطق للتدخل بناءً على البيانات التاريخية
- كشف الأنشطة المكررة أو المتداخلة

---

#### 14. 📡 API للتكامل الخارجي (REST API)
**المقترح:** بناء API endpoints لأهم البيانات (الأنشطة، المستفيدون، الإحصائيات) للتكامل مع أنظمة OCHA أو UN agencies.

---

#### 15. 🏷️ نظام الوسوم والتصنيف (Tags & Labels)
**المقترح:** إضافة `tags` مرنة للأنشطة (مثل: #طارئ، #متكرر، #ممول_خارجياً) لتسهيل الفلترة والبحث.

---

## 🏆 التوصية الأولى

> إذا كنت تريد **أقصى قيمة** في أقل وقت، أنصح بالبدء بـ:
>
> **نظام الإشعارات + تتبع تقدم الأنشطة + لوحة التحكم التنفيذية**
>
> هذه الثلاثة معاً ستحول النظام من "نظام تسجيل بيانات" إلى "منصة إدارة عمليات حقيقية".
